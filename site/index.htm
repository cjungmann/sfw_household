SFW Households Demo

This website demonstrates several implementations of a many-to-one interaction. The each list item below contains a link to an implementation along with a description of the progamming pattern used therein. A more complete description, along with installation instructions, are available as README.md files the links to which can be found in the main README.md on the project's main GitHub page.

  • Type 1 This pattern is the simplest, using fundamental built-in form- and table-based interactions. There are a few differences between a basic form/table setup:
    1. The MySQL person List procedure has a household filtering parameter in addition to the standard individual filtering parameter.
    2. The SRM household list mode changes the on_line_click URL to include the household record ID (to apply to the household filtering parameter) and household name (to make it available as a title for the table view).
    3. The SRM person list mode uses the household name off of the URL to put the household name in the page title to provide context.
